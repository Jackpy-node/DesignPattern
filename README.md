# DesignPattern

## 设计模式六大原则
1. 开闭原则：对扩展开放，对修改关闭
2. 里氏代换原则：任何基类可以出现的地方，子类一定可以出现
3. 依赖倒转原则：针对接口编程，依赖于抽象而不依赖于具体
4. 接口隔离原则：使用多个隔离的接口，比使用单个接口更好，降低类之间的耦合性
5. 迪米特原则（最少知道原则）：一个实体应尽量减少与其他实体之间的相互作用，使得系统功能模块相互独立
6. 合成复用原则：尽量使用合成/聚合的方式，而不是使用继承
